How to use the phones database

This documentation includes explaination and examples of how to interact with the phones database based on REST principles. This allows users to create, retrieve, update and delete phones in the database. This is done using the HTTP requests (GET, POST, PUT and DELETE). The phones Web API endpoints return JSON metadata about phones (brand, model, os, screensize, and image) directly from the database.
NOTE: the server runs using the command "npm start"

Create Retrieve Update Delete RetrieveAll Reset

Create a phone row

This method sends an HTTP post request to create a new phone in the database.
example:
URL call: http://localhost:3000/create
Request body (json):{"brand":"Apple","model":"iPhone","os":"IOS","image":"https://upload.wikimedia.org/wikipedia/commons/thumb/3/32/IPhone_X_vector.svg/440px-IPhone_X_vector.svg.png","screensize":6}

Possible Responses:

  • Success(201):Phone has been added to the database.
  • If an error is present in the request body (json) then error is shown in the console

Retrieve a phone

This method sends an HTTP get request to the database and retrieves the phone with the specified id.
example:
URL call: http://localhost:3000/retrieve/id

Possible Responses:

  • "Phone with id has been retrieved" is displayed in the console and the phone with matching id is returned in the response body (json).
  • Error 404: Phone with id was not found.
  • If a misspelling is present in the URL e.g./retriee/id then an error is returned ( Cannot GET /retriee/id)

Update phone metadata

This method sends an HTTP put request to the database and updates the phone metadata with the specified id.
example:
URL call: http://localhost:3000/update/id
Request body (json):{"brand":"Samsung","model":"S20"}

Possible Responses:

  • "Phone with id has been updated" is displayed in the console and returned in the response body.
  • Error 404: Phone with id was not found.
  • If a misspelling is present in the URL e.g./updaaate/id then an error is returned ( Cannot PUT /updaaate/id)

Delete a phone

This method sends an HTTP delete request to the database and deletes the phone with the specified id.
example:
URL call: http://localhost:3000/delete/id

Possible Responses:

  • "Phone with id has been deleted from the database" is displayed in the console and returned in the response body.
  • Error 404: Phone with id was not found.
  • If a misspelling is present in the URL e.g./delte/id then an error is returned (Cannot DELETE /delte/id)

Retrieve all phones

This method sends an HTTP get request to the database and retrieves all phones.
example:
URL call: http://localhost:3000/retrieveAll

Possible Responses:

  • "All phones in the database have been retrieved" is displayed in the console and all phones are returned in the response body (json).
  • if a misspelling is present in the URL e.g./retrieveAl then an error is returned (Cannot GET /retrieveAl)

Reset the database

This method sends an HTTP delete request to the database and deletes all the phones in the database.
example:
URL call: http://localhost:3000/reset

Possible Responses:

  • "All phones in the database are deleted" is displayed in the console and returned in the response body.
  • If a misspelling is present in the URL e.g./resit then an error is returned (Cannot DELETE /resit)

Safe and Idempotent

From the mentioned HTTP methods, the GET method is safe (GET does not alter the state of the database) and idempotent (GET could be called multiple times and will always get the same results).
Whereas, PUT, POST and DELETE are idempotent (they could be called multiple times and will always get the same results) but they are NOT safe (they alter the state of the database).
